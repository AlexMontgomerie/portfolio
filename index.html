Network Hardware Mapper

A project I worked on during my Internship with BT to map hardware in the network.

During my internship with BT, I was tasked to develop a tool for mapping physical connections between routers and other devices in the network by polling connection information from the hardware. Due to the heritage of BT, there is a lot of legacy equipment in the network and outside of documenting all the hardware and connections by hand, there is use for an automated tool which can track hardware live, and make this data more accessible to technicians who need it. By getting information from SNMP requests (using a Python library), I was able to compile a graph of hardware in a given network by iteratively polling new devices found. This backend tool was used in combination with a frontend UI written in JavaScript to display the nodes and edges in the network. The project helped me develop practical skills with Python as well as computing concepts such as running multiple threads to speed up polling multiple devices. I also gained familiarity with UNIX based systems and the internship gave me a very practical understanding of network design.

  • Date: September 2016

Braille Signer

A second year project to help with communication for deafblind people.

For a second year project, I was part of a group of students who proposed a solution to help with communication for deafblind people. The idea came about after researching current products available to deafblind people, and seeing how cumbersome and expensive they can be. We chose to design a portable and modern communicator that can easily interface with a phone or other communicators to ease communication for the user. The glove works by sending and receiving text through a BLE (Bluetooth Low Energy) interface. The messages are decoded to braille, with some parsing features to reduce common words into braille shorthand. The braille characters drive 6 vibrating motors, which indicate the character for the user. A set of buttons on the glove allow the user to send messages in return. A smartphone app was developed to demonstrate the device. This project allowed me develop my embedded system design skills significantly as I wrote low-level code on top of a Bluetooth protocol stack. It was also a chance to develop general group work skills, as different components of the project had to be integrated with the bluetooth controller, so there was lots of communication with different groups in the team.

Librainman

A project I worked on during my UROP with the Department of Computing at Imperial.

As part of my UROP with the Department of Computing at Imperial, I worked on a CNN acceleration project in conjunction with Corerain Technologies. My role within the team was to work on Hardware Verification and also wrote hardware drivers for the hardware modules. The CNN accelerator was designed on a Xilinx FPGA (Zynq SoC), and so a lot of the Hardware Verification tasks consisted of using Vivado Tools to check signals against simulated results and finding what caused the difference in results. For the hardware drivers, I used Xilinx tools to generate basic drivers for use with a linux system, and wrote and modified more code to create a simple interface with the hardware. I developed my skills in Vivado over the project and also developed an interest in CNN acceleration with FPGAs, as this is a new area of research and has a wide range of interesting applications.

HeadAid

An IoT project, part of a third year course.

HeadAid was a project to design a way of detecting concussions from high impact sports such as Ice Hockey and American Football. An accelerometer and gyroscope sensor placed inside a helmet were used to detect the magnitude of an impact as well as the direction of the force. The sensor readings are then passed through a clustering algorithm to determine which events lead to concussion, and immediately indicate to a coach or someone on the sideline that a player has a possible head injury. I mostly worked on the microprocessor for the hardware, and wrote MicroPython code to stream data captured from the sensors to a server to do processing, whilst trying to keep the power consumption as low as possible. This project was very interesting as it combined many different elements such as machine learning and embedded system design. This project also gave me a lot of skills in working within a team in a short timeline to produce an end product.
